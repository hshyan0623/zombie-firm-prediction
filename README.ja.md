# ゾンビ企業の1年先予測

**財務諸表から企業のゾンビ化を予測する機械学習パイプライン — 中日上場企業、2016〜2024年**

本プロジェクトは、中国・日本の上場企業の約5万件の「企業-年」データを用いて、企業が**翌年**にゾンビ化するかを予測する二値分類モデルを構築する。ケンブリッジ大学の修士論文（政府支援とシステミックリスクの実証研究）を基盤とし、そこで用いたゾンビ企業識別理論（FN-CHK法）を**予測特徴量**へ転用している。

> 🇬🇧 English: [README.md](README.md) ／ 🇨🇳 中文: [README.zh.md](README.zh.md)

## 「モデルを回すだけ」ではない設計

| 設計上の選択 | 何を示すか |
|--------------|-----------|
| **データリーク防止の目的変数** | ゾンビラベルは当年の財務から計算されるため、当年を予測すると答えが漏れる。ラベルを1年先へずらし（`t`の特徴量 → `t+1`のラベル）、真の予測を強制する。 |
| **ドメイン知識の特徴量化** | 論文の金利ギャップ理論（CHK法）を予測特徴量へ。計量経済学とMLの架橋。 |
| **クラス不均衡** | 正例（ゾンビ）は約8%。`class_weight` / `scale_pos_weight`で対応し、正解率ではなく**PR-AUC**で評価。 |
| **時系列分割** | ランダム分割ではなく、過去（2016–2020）で学習し未来（2021–2022）を検証。実運用を模倣。 |
| **説明可能性** | SHAPでどの財務シグナルが予測を駆動するか、経済理論と整合するかを検証。 |
| **エラー分析** | 偽陽性・偽陰性を分解し、モデルが「どこで・なぜ」誤るかを理解。 |

## 主要な結果

| モデル | AUC | PR-AUC |
|--------|-----|--------|
| ロジスティック回帰（ベースライン） | 0.927 | 0.591 |
| **ランダムフォレスト（最良）** | **0.954** | **0.672** |
| XGBoost（調整後） | 0.952 | 0.663 |
| ランダム基準 | 0.500 | 0.095 |

PR-AUC 0.67 はランダム基準（0.095）の**約7倍**であり、1年先のゾンビ化を実用的な精度で予測できる。

### 指標を超えた発見

1. **特徴量 > モデルの複雑さ。** 理論に基づく特徴量設計により、単純なロジスティック回帰でもPR-AUC 0.59に達する。複雑なモデルの上乗せは小さい。この問題ではモデル選択よりドメイン知識に基づく特徴量が重要。
2. **理論をMLが裏づけ。** SHAPでは資産負債率とROAが最上位、利息系（実効金利・**金利ギャップ(CHK)**・カバレッジ）が上位6件に入る。経済学とMLが一致。
3. **ゾンビ化は国境を越える。** 国籍（`is_china`）のSHAP重要度は低く、ゾンビ企業の財務的特徴は概ね国境を超えて共通。
4. **モデルの盲点が示唆的。** 見逃したゾンビ（偽陰性）は「表面上は財務改善だが、なお隠れた支援に依存する企業」に集中し、金利ギャップのような本質的特徴量の必要性を裏づける。

## 手法の概要

**目的変数。** 各「企業-年」`t`について、その企業が`t+1`年にゾンビ（FN-CHK）かどうかをラベルとする。ゾンビ識別は金利ギャップ条件に収益性・レバレッジ基準を組み合わせ、国別の政策金利（中国：人民銀行、日本：日銀平均約定金利）を用いる。

**5分類18特徴量。** 負債構造（レバレッジ・期限構造）、利息・返済能力（実効金利・**金利ギャップ**・カバレッジ）、収益性（ROA・赤字フラグ）、動的トレンド（各種成長率・連続赤字年数・ROAボラティリティ）、国籍。すべて比率のため企業規模によらず比較可能。外れ値は inf→NaN と1%/99% Winsorization で処理。

**モデル。** ベースラインにロジスティック回帰（中央値補完＋標準化）、主モデルにランダムフォレストとXGBoost（欠損を自然に扱える）。不均衡は重み付けで対応し、ハイパーパラメータは**訓練データ内の交差検証のみ**でグリッドサーチ調整。

## 再現方法

```bash
pip install -r requirements.txt
jupyter notebook notebooks/zombie_prediction.ipynb
```

- **Part 1〜3**（生データ処理）はライセンス元データが必要で、単体では実行不可。
- **Part 4〜7**（モデリング・SHAP・エラー分析・調整）は同梱の `data/ml_dataset_features.csv` から直接実行可能。

---

## データとプライバシー

企業レベルの生データ（中国：CSMAR、日本：Osiris）は専有データのため**非公開**。匿名化した特徴量のみ同梱：企業識別子は不透明なコード（例 `CN_000001`）、値はすべて計算済み比率で、企業名も生の財務額も含まない。
